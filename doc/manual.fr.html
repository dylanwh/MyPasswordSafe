Introduction

MyPasswordSafe est un gestionnaire de mots de passe simple a utiliser qui maintient la compatibilité avec les fichiers utilisé par Password Safe. MyPasswordSafe offre les fonctions suivantes:

  • Les coffres-forts sont chiffrés lorsqu'ils sont stockés sur disque.
  • Les mots de passe n'ont jamais a être vus car ils sont copiés dans le presse-papier
  • Des mots de passe aléatoires peuvent être générés.
  • Les taille, position et largeurs des colonnes des fenêtres sont mémorisées.
  • Un coffre-fort peut être rendu actif de telle sorte qu'il soit toujours ouvert lorsque MyPasswordSafe démarre.

Installation

MyPasswordSafe existe en deux distributions: une distribution binaire et une  distribution source. MyPasswordSafe nécessite Qt 3 et Boost, donc vérifiez que vous en disposez. Boost n'est requis que pour compiler MyPasswordSafe, mais Qt a besoin d'être installé pour exécuter les binaires.

...Depuis les binaires

La distribution binaire est précompilée. Elle peut être téléchargée depuis cette page. Une fois téléchargée, "détarez" la:

tar xvfz MyPasswordSafe-bin.tgz

Cela devrait créer un répertoire contenant le binaire MyPasswordSafe, ce fichier, et d'autre documentation. Lancer "make install" dans le répertoire MyPasswordSafe directory va tout installer sous /usr/local. Cela devrait fonctionner dans d'autres endroits, mais les fichiers de langue ne seront pas trouvés  s'ils ne sont pas dans /usr/local/share/MyPasswordSafe/locale. Cela devrait être tout si ça fonctionne. Sinon continuez à lire.

...Depuis les sources

Vous voulez donc compiler vous même. Assurez vous que Qt 3 est installé, et téléchargez Boost. Récupérez les sources et "détarez" MyPasswordSafe: 

tar xvfz MyPasswordSafe.src.tgz

Vous pouvez "détarer" Boost dans le répertoire de  MyPasswordSafe. Le fichier projet (MyPasswordSafe.pro) demande "boost-1.30.2" dans le répertoire. Vous pouvez changer le fichier projet à la main si votre chemin vers boost est différent. Le Makefile contient également une variable PREFIX qui détermine l'endroit où MyPasswordSafe s'installera. Changez la si /usr/local n'est pas l'endroit où vous voulez qu'il s'installe. Enfin vous pouvez lancer make: 

cd MyPasswordSafe
make
make install

En cas de succès vous pouvez maintenant commencer à utiliser MyPasswordSafe. Sinon envoyez moi un email décrivant votre situation. Au cas où vous souhaiteriez désinstaller MyPasswordSafe, le Makefile a une  directive uninstall. Lancer "make uninstall" effacera les fichiers installés.

Usage

MyPasswordSafe est très simple à utiliser. Presque toutes les commandes sont accessibles depuis le menu, la barre d'outils, le menu contextuel ou le clavier. Avant d'aborder l'utilisation proprement dite, regardons comment MyPasswordSafe peut être démarré.

Démarrage de MyPasswordSafe

Le démarrage de MyPasswordSafe est semblable à celui de n'importe quel autre programme, vous avez juste à le lancer. Si vous avez copié le binaire dans votre PATH, tapez simplement MyPasswordSafe. Sinon vous devez soit vous positionnez dans le répertoire où se trouve le binaire, soit inclure celui-ci dans la ligne de commande. Vous pouvez également désirer ajouter une icône dans votre lanceur favori.

Lorsque MyPasswordSafe démarre pour la première fois, un coffre-fort vide est créé. Comme vous l'apprendrez plus tard, MyPasswordSafe possède une chose nommée le coffre-fort actif. Le coffre-fort actif est ouvert sans que vous ayez besoin de le localiser encore et encore.

La fenêtre principale

La fenêtre principale n'est pas vraiment complexe. Il y a une boîte à liste, une barre d'outils, et un menu. La barre d'outils et le menu font un peu double emploi. La plupart de l'action se déroule dans la boîte à liste.

La boîte à liste contient les mots de passe du coffre-fort courant. Double-cliquer sur un mot de passe dans la boîte à liste récupère le mot de passe de cette entrée, et cliquer-droit fait surgir le menu d'édition à côté du pointeur. Alors que font ces commandes?

Gestion des mots de passe

Le menu d'édition contient les commandes utilisées pour gérer les mots de passe. Le nom de chaque commande décrit ce qu'elle fait. Par exemple, Ajouter Mot de passe ajoute réellement un nouveau mot de passe. Supprimer Mot de passe supprime le mot de passe sélectionné. Et Editer Mot de passe fait surgir un dialogue pour éditer le mot  de passe sélectionné.
Les deux dernières commandes nécessitent peut-être des explications. Ce sont Récupérer Utilisateur et Récupérer Mot de passe. Toutes les deux copient de l'information vers le presse-papier. L'une copie l'utilisateur, l'autre copie le mot de passe.

Dialogue d'édition des mots de passe

Et Ajouter Mot de passe et Editer Mot de passe utilisent le même dialogue pour éditer les données d'un mot de passe. Les mots de passe possèdent quatre champs d'information: nom, utilisateur, mot de passe et notes. Tous sauf le champ mot de passe sont affichés dans la fenêtre principale. Le dialogue possède également trois boutons qui affectent le champ mot de passe:

  • Montrer: Rend visible le mot de passe
  • Récupérer: Copie le mot de passe dans le presse-papier.
  • Générer: Remplis le champ mot de passe avec un mot de passe aléatoire.

Ouverture et sauvegarde des coffres-forts

Les coffres-forts peuvent être créés, ouverts, et sauvegardés depuis le menu Fichier. Les commandes fonctionnent comme dans n'importe quelle application sauf lorsque qu'une phrase de passe est requise.
MyPasswordSafe supporte de multiples formats de fichier. Actuellement seuls deux formats sont supportés: les coffres-forts Password Safe et les fichiers texte.
Password Safe est similaire à MyPasswordSafe, mais ne tourne que sous Windows. Les fichiers créés par Password Safe peuvent être ouverts par MyPasswordSafe et vice-versa. Les fichiers sont chiffrés avec l'algorithme Blowfish et sont présumés sûrs. Utiliser dat comme extension.

Les fichiers texte ne devraient JAMAIS être utilisés. Toute l'information est stockée non chiffrée. Mais si vous les trouvez utiles, allez-y et utilisez les.

Préférences

MyPasswordSafe n'est pas des plus configurables mais offre quelques options:

  • Coffre-fort actif: C'est le coffre-fort que MyPasswordSafe ouvre quand il démarre; il peut également être spécifié via Activer dans le menu Fichier.
  • Utilisateur par défaut: C'est l'utilisateur utilisé pour les nouveaux mots de passe.
  • Vider le presse-papier en sortant?: Si coché, le presse-papier est vidé quand MyPasswordSafe est fermé.
  • Longueur des mots de passe générés: Fixe la longueur des mots de passe générés en appuyant sur le bouton Générer dans le dialogue Editer Mot de passe.
  • Nombre d'essais: Nombre de fois que MyPasswordSafe essayera d'obtenir de vous la phrase de passe correcte avant d'abandonner.

MyPasswordSafe stocke également la position et la taille de la fenêtre ainsi que les largeurs des colonnes. Elles sont fixées en les changeant, et ne sont pas disponibles dans le dialogue Préférences.

FAQ

Quelle est la sécurité de MyPasswordSafe?

Les coffres-forts devraient être sûrs sauf si l'algorithme Blowfish est cassé. C'est lorsque le programme tourne que je suis incertain de sa sécurité. Je suis sûr qu'un programme embêtant pourrait trouver vos mots de passe en mémoire.
Si vous savez quoi que ce soit pour rendre ce programme à l'épreuve des balles, envoyez moi un mail ou un correctif.

Où est la version Windows?

J'aimerais réellement qu'il y ait une version Windows mais je ne peux me permettre une licence Qt. Si vous avez besoin d'un gestionnaire de mots de passe pour Windows, récupérez Password Safe. Les versions < 2 créent des fichiers que MyPasswordSafe peut charger et sauvegarder. [ Note pour les développeurs: Si vous avez Qt pour Windows, pourriez vous le compiler et le packager pour moi? ]
